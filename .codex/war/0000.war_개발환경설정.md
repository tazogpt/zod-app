# 목적

Spring Boot + Kotlin 백엔드의 초기 부트스트랩을 수행한다.
- 최소 Gradle 의존성 설정
- 기본 `application.yml` 작성

---

# 프로젝트 전제

- 빌드 도구: Gradle (Kotlin DSL)
- Gradle 실행: 반드시 Wrapper(`./gradlew`) 기준
- Java: Temurin 25
- Kotlin
- Spring Boot
- 멀티 모듈 구조 (app / war 중 war 기준)
- 설정 파일:
  - `war/src/main/resources/application.yml`
  - `war/src/main/resources/application.yml`에서 `application-dev.yml`을 import (app 모듈 리소스)

---

# Gradle 규칙

## 1. 실행 기준
- 시스템 Gradle 설치 여부를 전제로 하지 않는다
- 모든 설명과 코드는 `./gradlew` 기준으로 작성한다

## 2. 의존성 원칙
- 최소한의 필수 모듈만 추가한다
- 사용하지 않을 가능성이 있는 스타터는 추가하지 않는다
- actuator, security, test 확장 등은 이후 단계에서 다룬다

### 허용되는 기본 의존성 범위
- spring-boot-starter
- spring-boot-starter-web
- spring-boot-starter-thymeleaf
- project(:app) (war에서 app 모듈 참조 시)

---

# Spring Boot 설정

- `bootJar` 비활성화, `bootWar` 활성화
- `mainClass`는 `zod.WebApplication` 기준으로 지정

---

# application.yml 작성 규칙

## 1. 역할
- 기본 실행에 필요한 최소 설정만 작성한다
- 프로파일 분기, 외부 시스템 연동 설정은 넣지 않는다

## 2. 허용 범위
- `spring.application.name` 수준의 기본 식별 정보
- `spring.datasource` 및 `spring.jpa`의 기본 설정 (import 대상에 포함 가능)
- `spring.thymeleaf` (템플릿 설정: cache, check-template-location, encoding, mode)
- `server.port`, `server.servlet.context-path` (MVC 기본 설정)
- 환경별 설정은 이후 단계에서 추가한다

## 3. 추가 설정
- `spring.main.web-application-type: servlet`

---

# 현재 구성

- `spring.application.name: zod`
- `spring.config.import: application-dev.yml`
- `spring.main.web-application-type: servlet`
- `spring.thymeleaf`: `cache=false`, `check-template-location=false`, `encoding=UTF-8`, `mode=HTML`
- `server.port: 18080`, `server.servlet.context-path: /`

---

# 작업 순서

1. `build.gradle.kts`에 최소 Gradle 플러그인 및 의존성 정의
2. Kotlin / Spring Boot 버전 명확히 지정
3. `src/main/resources/application.yml` 기본 설정 작성

---

# Codex 작업 지시

- 이 문서를 작업 기준으로 삼아 판단한다
- 불필요한 라이브러리 제안은 하지 않는다
- “나중에 필요할 수 있다”는 이유로 의존성을 추가하지 않는다
- 결과는 즉시 실행 가능한 상태여야 한다
- 추측 대신 명시적인 근거가 없는 설정은 제외한다

---

# 산출물 기대 형태

- Gradle 설정: 간결하고 명확
- application.yml: 최소 설정만 포함
- 이후 단계(보안, 설정 분리, 멀티 모듈 확장)가 가능한 상태
