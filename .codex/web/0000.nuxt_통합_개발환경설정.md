# 목적

Frontend Nuxt 통합 개발 환경을 설정한다.
- admin/partner/site 모듈 초기 셋업 기준 정리
- 최소 의존성 및 실행 기준 정리
- 기본 실행/빌드 명령 기준 정의

---

# 프로젝트 전제

- 프론트엔드 모듈: `web/admin`, `web/partner`, `web/site`
- 프레임워크: Nuxt
- 패키지 매니저: pnpm
- 패키지 매니저/스크립트는 해당 모듈 기준으로 실행

---

# 작업 규칙

- 불필요한 의존성은 추가하지 않는다
- 프레임워크 기본 규칙을 우선한다
- 설정 변경 시 해당 문서에 기록한다

---

# 작업 순서

1. `web/admin`, `web/partner`, `web/site` 기준으로 초기 설정 확인
2. 필요한 의존성/스크립트 최소 추가
3. 기본 실행/빌드 확인

---

# 현재 구조

각 모듈(`web/admin`, `web/partner`, `web/site`)은 동일한 Nuxt 초기 구조를 가진다.
- `app/app.vue`
- `nuxt.config.ts`
- `package.json`
- `pnpm-lock.yaml`
- `public/`
- `tsconfig.json`

---

# Codex 작업 지시

- 이 문서를 작업 기준으로 삼아 판단한다
- 추측 대신 명시적인 근거가 없는 설정은 제외한다
- 추가 설정이 필요한 경우 문의를 해서 해당 부분을 채워 나간다

---

# 산출물 기대 형태

- 최소 설정으로 실행 가능한 상태
- 이후 확장이 가능한 구조

---

# 권한 문제로 Codex 에서 자동으로 작업할수 없다.
```bash
pnpm create nuxt@latest web/admin
pnpm create nuxt@latest web/partner
pnpm create nuxt@latest web/site
```
