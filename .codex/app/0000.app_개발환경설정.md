# 목적

Spring Boot + Kotlin 백엔드의 초기 부트스트랩을 수행한다.
- 최소 Gradle 의존성 설정
- `Application.kt` 작성
- 기본 `application.yml` 작성

---

# 프로젝트 전제

- 빌드 도구: Gradle (Kotlin DSL)
- Gradle 실행: 반드시 Wrapper(`./gradlew`) 기준
- Java: Temurin 25
- Kotlin
- Spring Boot
- 멀티 모듈 구조 (app / war 중 app 기준)
- 설정 파일:
  - `app/src/main/resources/application.yml` (운영/기본)
  - `app/src/test/resources/application.yml` (테스트)

---

# Gradle 규칙

## 1. 실행 기준
- 시스템 Gradle 설치 여부를 전제로 하지 않는다
- 모든 설명과 코드는 `./gradlew` 기준으로 작성한다

## 2. 의존성 원칙
- 최소한의 필수 모듈만 추가한다
- 사용하지 않을 가능성이 있는 스타터는 추가하지 않는다
- actuator, security, test 확장 등은 이후 단계에서 다룬다

### 허용되는 기본 의존성 범위
- spring-boot-starter
- spring-boot-starter-web
- spring-boot-starter-webflux
- spring-boot-starter-data-jpa
- jackson-module-kotlin ㅇ
- kotlin-reflect
- kotlin-stdlib
- spring-boot-starter-test (Application 기본 테스트 작성 시)
- mssql-jdbc (MS-SQL 사용 시)

# Application.kt 작성 규칙

## 1. 역할
- `zod.Application.kt`는 애플리케이션 시작점만 담당한다
- 비즈니스 로직, 설정 클래스, 컴포넌트 스캔 커스터마이징을 넣지 않는다

## 2. 구조
- `@SpringBootApplication` 사용
- `main` 함수는 표준 형태 유지

## 3. 금지 사항
- 임의의 설정 코드 삽입 금지
- 임시 테스트 코드 추가 금지
- 로그 설정, 프로파일 분기 금지

---

# application.yml 작성 규칙

## 1. 역할
- 기본 실행에 필요한 최소 설정만 작성한다
- 프로파일 분기, 외부 시스템 연동 설정은 넣지 않는다

## 2. 허용 범위
- `spring.application.name` 수준의 기본 식별 정보
- `spring.datasource` 및 `spring.jpa`의 기본 설정
- 환경별 설정은 이후 단계에서 추가한다

## 3. 테스트 설정
- 테스트는 `src/test/resources/application.yml`로 분리한다
- 기본값은 MS-SQL 접속 정보를 사용한다

---

# 작업 순서

1. `build.gradle.kts`에 최소 Gradle 플러그인 및 의존성 정의
2. Kotlin / Spring Boot 버전 명확히 지정
3. `src/main/kotlin/zod/Application.kt` 작성
4. `src/main/resources/application.yml` 기본 설정 작성
5. `src/test/resources/application.yml` 기본 설정 작성
6. `./gradlew bootRun` 또는 `./gradlew build` 기준으로 정상 빌드 확인

# Codex 작업 지시

- 이 문서를 작업 기준으로 삼아 판단한다
- 불필요한 라이브러리 제안은 하지 않는다
- “나중에 필요할 수 있다”는 이유로 의존성을 추가하지 않는다
- 결과는 즉시 실행 가능한 상태여야 한다
- 추측 대신 명시적인 근거가 없는 설정은 제외한다

---

# 산출물 기대 형태

- Gradle 설정: 간결하고 명확
- Application.kt: 표준적이고 단순
- application.yml: 최소 설정만 포함
- 이후 단계(보안, 설정 분리, 멀티 모듈 확장)가 가능한 상태
